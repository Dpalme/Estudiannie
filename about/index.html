About QuizDb

Esta aplicación tiene como objetivo facilitar a los estudiantes la manera en que pueden crear flashcards y poder estudiar más fácilmente. Se pueden crear diferentes grupos de preguntas y clasificar las preguntas dentro de esos grupos como los usuarios quieran. Un ejemplo de eso sería poder clasificar las preguntas por materia y así poder estudiar para la materia que sea necesario. Una vez elegido el grupo el usuario puede solicitar una pregunta elegida aleatoriamente con su respectiva respuesta.

Base de Datos

IndexedDB es una base de datos transaccional como cualquier base de datos relacional basada en SQL, indexedDB es una base de datos basada en JavaScript la cual maneja objetos JSON de una BD NoSQL, tiene como ventaja la facilidad de mapear objetos completos a pesar de que no se cuente con sus atributos totalmente definidos.

Las operaciones de IndexedDB son asíncronas, esto con el objetivo de no bloquear aplicaciones. Sin embargo, si se necesita la implementación síncrona se piensa reintroducir la funcionalidad si los desarrolladores lo requieren.

IndexedDB usa Object Stores en lugar de lo que normalmente conocemos como tablas. Para QuizDb, usamos 2 object stores, list_ref y questions.

La aplicación usa Bootstrap 4.1 y Peoria.js para lograr su interfaz minimalista, dinámica y contenida en una página.

Modelo

En list_ref se guarda el nombre de los diferentes grupos de preguntas mientras que en questions se guardan las preguntas.

list_ref
El object store list_ref es el encargado de mantener el registro de todos los diferentes grupos de preguntas que están en la base de datos.

Contiene únicamente una columna “name”, el string tiene que ser único y por tanto, no contiene índices numéricos.

questions
Contiene cuatro columnas. id, question, answer y group.
id: Valor numérico por el cual se ordenan las preguntas almacenadas.
question: string no único
answer: string no único
group: nombre del grupo de preguntas al que pertenece la pregunta
